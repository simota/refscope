### Mira — Senior Product Designer

#### Request 1: Lens 群を「単一空間」に統合する spatial map / Lens taxonomy

**Speaker:** Mira (Senior Product Designer × Linear-comparison)
**Scene:** 月曜の朝、Linear の issue list を左 monitor、Refscope を右 monitor に置く。30 分かけて Pulse / Drift / Hotspot / Risk-Heatmap / Risk-Trend / CoChange / Outbox / Digest / FileStream を順に切り替えたあと、彼女は Notion の design diary に書く: 「Refscope は賢いけど、見ていて落ち着かない。レンズ 11 個を 11 個別々の "部屋" として渡されている。」

##### User Voice (First Person)
> "Linear なら『1 つの空間の中で視点が動く』感じだけど、Refscope は『別々の部屋に何度も入り直す』感じ。私は LensSwitcher を開くたびに、自分が今どのレンズを見ていたか、これから見るレンズが今のレンズの隣なのか向かいなのか、思い出せない。Lens 名を覚えるんじゃなくて、空間のどこに何があるかを覚えたい。Lens は **増やすほど偉い** ものじゃなくて、**配置で意味を語る** ものだと思う。"

##### Why This Is Needed
- Lens が 11 枚を超えると線形リストでは visual orientation が破綻する
- LensSwitcher は単なる "切替装置" として強くなり、ユーザーが Lens 名と機能の mapping を毎回脳内で再構築している
- Refscope の brand metaphor (instrument / scope) は「装置のどこを覗くか」という空間メタファーに自然に親和するのに、UI ではフラットな switch list になっている

##### Acceptance Criteria (User Perspective)
- [ ] Lens 間に明示的な階層 or グルーピング (例: 観測 / 解釈 / リスク / 意思決定) が見える
- [ ] 切り替え時、Lens 同士の "距離" が motion 2-3px で示唆される (`Motion is for state change only` 原則と整合: 状態遷移そのもの)
- [ ] LensSwitcher を畳んだ状態でも、現在の Lens がどの群に属するかが badge / breadcrumb / sidebar position で読める
- [ ] 新規 Lens 追加時、どの群に属するかを宣言しないと merge できない (lint or PR template)

##### Emotional Impact
- **Current emotion:** 警戒 (落ち着かない / 自分の現在地が読めない)
- **Post-fulfillment emotion:** 安堵 (空間として読める)
- **User-felt urgency:** 毎朝の inconvenience (Lens 切替を 1 日 30 回以上)

#### Request 2: DetailPanel の "flat density" を 3-layer typography rhythm に立て直す

**Speaker:** Mira (Senior Product Designer × Linear-comparison)
**Scene:** 既知の force-push の現場を再現するため、適当な commit を選んで DetailPanel をスクロール。commit hash, message, author, file list, structural diff badge, signature status, why-panel が縦に並ぶ。Mira は声に出す:「全部、同じ重さで降ってくる。」

##### User Voice (First Person)
> "DetailPanel は密度はあるけど、密度には『階層付きの密度』と『フラットな密度』がある。今の Refscope は後者。commit hash は事実、message は author の意図 (=人の解釈)、file list は派生情報、signature status はメタ情報。layer がぜんぶ違うのに、同じカード内で同じ density で並んでる。私は **視線をどこに置くか** を毎回選ばされる。calm by default の brand ならここを直してほしい。"

##### Why This Is Needed
- visual-direction.md は Density principle を明示しているが、DetailPanel 内の typography hierarchy が "title / body / meta" の 3 段階どまり

##### Acceptance Criteria (User Perspective)
- [ ] DetailPanel に明示的な primary / secondary / tertiary / meta の 4-layer typography rhythm
- [ ] 各レイヤーは line-height / letter-spacing / weight / 余白 の組合せのみで識別可能 (color に依存しない — 三層保証原則と整合)

- [ ] モノクロ印刷しても 4 layer が読み分けられる (visual-direction の "リトマス試験紙" を AC として明文化)

##### Emotional Impact

### Saoirse — Inclusive / Accessibility Designer

#### Request 5: WCAG 2.2 AA conformance audit と reduced-motion / forced-colors の first-class 実装

**Speaker:** Saoirse (Inclusive Designer × reduced-motion + WHCM)
**Scene:** 朝の macOS 環境では Refscope は美しく動く。夕方、検証用 Windows 機で Edge を開き、Windows High Contrast Mode をオン。Lens の status badge が消える。reduced-motion を全 OS で有効にしているのに、live update の pulse がチカチカ見える瞬間がある。

##### User Voice (First Person)
> "calm by default は素晴らしい brand axis。でもそれは『多数派にとっての calm』。私には reduced-motion をかけても、SSE 由来の DOM 更新が **CSS animation でない経路** で動いてる瞬間が見える。Windows の forced-colors mode で開いたら、accent cyan は消えるし status badge の意味色も消えて、何が rewritten で何が新規かが border でしか伝わってこない。**WCAG 2.2 AA を一度きちんと audit してほしい。** focus visible は accent cyan じゃなくて system 標準を使うか、両方サポートしてほしい。Refscope は『観測装置』を名乗る以上、**観測者がどんな device / impairment であっても観測できる** べき。"

##### Why This Is Needed
- 過去 round で Yuki (motion) / Lin (color-blind) が部分的にカバーしたが、**motion-reduction × forced-colors × focus-visible × tab order** の 4 軸組合せ audit は未実施
- visual-direction.md の "Motion is for state change only" は紙の上の原則で、実装層で `prefers-reduced-motion: reduce` 下に **全 motion が 0ms に倒れる** ことは保証されていない
- forced-colors mode (Windows High Contrast / forced-colors media query) で意味色 token が消える時の fallback 設計が見えない
- focus-visible は visual-direction で accent cyan に寄っている可能性 — WHCM 下では Canvas-anchored focus が必要

##### Acceptance Criteria (User Perspective)
- [ ] `prefers-reduced-motion: reduce` 下で全 transition / pulse / shimmer / live-update animation が即時遷移 (`animation-duration: 0.01ms` でなく構造的に skip)
- [ ] `forced-colors: active` で全 status badge が text + icon + outline で読める (border 強調 / system color tokens に bind)
- [ ] focus-visible style は system focus を尊重 (WHCM 下は CanvasText / system focus に追従、通常時は accent cyan で OK)
- [ ] axe-core / Lighthouse + WCAG 2.2 SC checklist (1.4.11 Non-text Contrast / 2.2.6 Timeouts / 2.5.7 Dragging / 3.2.6 Consistent Help) を CI artifact として残す
- [ ] keyboard trap / tab order audit を Lens 切替・CommandPalette・FileContextMenu・RewriteRescuePanel で実施

##### Emotional Impact
- **Current emotion:** 諦めと毎回の再 confirm 疲れ
- **Post-fulfillment emotion:** 信頼 ("この製品はちゃんと自分も観測者扱いしてる")
- **User-felt urgency:** **access-blocking — 毎日の cumulative**
