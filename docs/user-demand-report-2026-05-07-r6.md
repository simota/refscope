## Questions for the Team（前提への反証）

1. **「コミット粒度の diff があれば十分」という前提** — 実際にはリファクタ中のユーザーは「シンボル粒度・ファイル横断・意味等価性」で見たがっている。コミットを最小単位にしている UX は本当にリファクタユーザーに合っているか？

2. **「branch health の 4 状態（active/stale/merged/diverged）で十分」という前提** — Tech Lead は "並行する 6 本のリファクタブランチ" を束で見たい。バイナリ的フィルタではなく "リファクタ群ヘルス" のような集約ビューが必要では？

3. **「refscope は read-only ツール」というドメインを守りつつ、リファクタ事故からの "復元コマンド提示"（実行ではなく）まで踏み込むかどうか** — read-only の純度と、リファクタユーザーの最大のペイン（rewrite 事故）の救済は両立できるか、それとも明示的に範囲外と宣言すべきか？

4. **AST/semantic diff は refscope の "ローカル・依存最小" 哲学と両立するか** — tree-sitter を抱えるか、言語非依存の構造ヒューリスティクス（行対称性・トークン保存率）に留めるか。
